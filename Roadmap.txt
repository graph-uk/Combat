Пофиг на session-reuse. Каждый тест заново открывает браузер, и закрывает его за собой.

Каждый тест это самостоятельная программа, которая запускается с параметрами.
Возвращает 0 если всё хорошо, и короткую текстовую причину отказа (стектрейс) если всё плохо. Возвращаемым нулём-определяется успешность выполнения теста.
	Опционально, может создавать зипник <testname.zip> с тяжёлыми артефактами отказа (скриншоты, видео, дампы, etc..)
	Опционально - может отсылать тяжёлые артефакты отказа (скриншоты) на сервак, а в лёгком отчёте - указывать ссылки.
	Опционально - может отсылать на сервак статистику выполнения вне зависимости от успеха (время выполнения, количественные показатели)

Все параметры - строки. Произвольные, либо перечисляемые. Приведение к типу и проверка корректности значений - задача теста.
	Например:
		Произвольная - Timestamp (целое число).
		Перечисляемая - локаль ("CA","RU","US","UK")
	Примерный набор параметров некоего теста, который исполняет что-то от админа:
		SeleniumSession="./session.bin" 		//Имя файла, содержащего сессию
		Timestamp="" 							//Timestamp сессии. Используется для гнерации уникальных логинов, etc.
		Locale="RU" 							//Локаль, которую тестируем
		AdminPass=""							//Пароль админа. 
	У разных тестов может быть разное количество параметров. При запуске теста обязательно передавать все параметры со значениями. 
	"Лишние" параметры - игнорируются.
	Каждый тест умеет вернуть список поддерживаемых им параметров в JSON формате, если его вызвать без параметров.
		Опционально для тестов написанных на GO - есть готовая функция которая ищет исходник файла, и в нём находит параметры.
		ParamName 			ParamType 	Enum
		SeleniumSession 	var 		""
		SessionTimestamp 	var 		""
		Locale 				enum 		"CA,RU,US,UK"


Менеджер тестов принимает параметры, теги, имена, по которым будет фильтровать варианты исполнения теста.
Менеджер тестов находит все тесты, получает поддерживаемые параметры, теги. Если получить не удаётся - исключение.
Формирует список всех вариантов исполнения всех тестов.
Фильтрует список (оставляет только те варианты, которые удовлетворяют параметрам, который получил менеджер).
	Например тест поддерживает локали "CA","RU". Значит он будет запущен 2 раза. Будет 2 отчёта вида:
	<TestName>.201610102111.CA.txt
	<TestName>.201610102111.RU.txt
Запускает все тесты последовательно.
